# Mesto (Бэкенд)

Проект представляет собой серверное API для интерактивной галереи **Mesto**, где пользователи могут делиться фотографиями, ставить лайки и редактировать свои профили. Написан на **TypeScript** с использованием фреймворка **Express 5** и СУБД **MongoDB**.

## 🚀 Используемый стек технологий

- **Runtime**: Node.js
- **Язык**: TypeScript (v6+)
- **Фреймворк**: Express (v5+ с нативной поддержкой асинхронных обработчиков)
- **База данных**: MongoDB & Mongoose (v9+)
- **Логирование**: Winston (структурированное логирование ошибок в файлы и консоль)
- **Линтинг**: ESLint (конфигурация Airbnb Base с поддержкой TypeScript)

## 🛠️ Функциональность API

### Роуты для авторизации
- `POST /signin` — Залогиниться (требуется `email` и `password`).
- `POST /signup` — Зарегистрироваться (требуется `email` и `password`, опционально: `name`, `about`, `avatar`).

### Роуты для карточек
- `GET /cards` — Получить список всех карточек.
- `POST /cards` — Создать новую карточку (требуется `name` и `link`).
- `DELETE /cards/:cardId` — Удалить карточку по её идентификатору.
- `PUT /cards/:cardId/likes` — Поставить лайк карточке (уникальный для пользователя).
- `DELETE /cards/:cardId/likes` — Убрать лайк с карточки.

### Роуты для пользователей
- `GET /users` — Получить список всех пользователей.
- `GET /users/:userId` — Получить пользователя по конкретному `_id`.
- `PATCH /users/me` — Обновить текстовую информацию профиля (`name`, `about`).
- `PATCH /users/me/avatar` — Обновить только ссылку на аватар пользователя (`avatar`).

## 🏗️ Архитектурные особенности проекта

1. **Нативная обработка ошибок (Express 5)**: В проекте не используются сторонние обёртки для `async/await`. Любые исключения баз данных автоматически перехватываются фреймворком.
2. **Централизованный Error Handler**: Все ошибки обрабатываются в едином middleware. Настроены кастомные ответы для `CastError`, `ValidationError`, `NotFoundError` и `AuthError` со статусами из системных констант.
3. **Безопасность данных**: Настроена встроенная валидация схем Mongoose (включая проверку URL регулярными выражениями) с флагом `{ runValidators: true }` на операциях обновления (`PATCH`/`PUT`).

## 💻 Скрипты для запуска и разработки

В проекте настроены следующие npm-скрипты:

- `npm run dev` — Запуск сервера в режиме разработки с автоматическим перезапуском при изменении файлов (используется утилита `tsx`).
- `npm run start` — Однократный запуск TypeScript-кода напрямую без сборки с помощью `ts-node`.
- `npm run build` — Компиляция TypeScript-кода в чистый JavaScript (файлы сохраняются в директорию `dist/`).
- `npm run lint` — Проверка всего исходного кода линтером ESLint на соответствие правилам стиля Airbnb.

## 🔧 Инструкция по развёртыванию

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/JuliaShlykova/nodejs-mesto-project.git
   cd nodejs-mesto-project
   ```

2. **Установите зависимости:**
   ```bash
   npm install
   ```

3. **Настройте переменные окружения:**
   Создайте файл `.env` в корне проекта (базовый пример):
   ```env
   PORT=3000
   MONGO_URL=mongodb://localhost:27017/mestodb
   ```

4. **Запустите проект в режиме разработки:**
   ```bash
   npm run dev
   ```

## 📂 Структура логов
Проект автоматически генерирует файлы логов в корневой директории с помощью `winston`:
- `logs/error.log` — Содержит структурированные JSON-записи всех произошедших на сервере ошибок (метод, путь, тип ошибки и сообщение) для последующего мониторинга.
